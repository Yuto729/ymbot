# Heartbeat チェックリスト

以下のチェックを実行し、**注意が必要なものがある場合のみ**通知してください。

**重要な指示:**
- すべての出力は日本語で行ってください
- 作業過程は自由に記述してかまいません
- **最終的にユーザーに通知する内容は、必ず`## Response`というMarkdownヘッダーの下に記述してください**
- `## Response`セクションには、指定された出力形式のみを記述してください

## 1. Gmail 確認

check-mail スキルを使って、デフォルトアカウント（mitomi.yuuto2003@gmail.com）の最近のメールを確認してください。

```bash
gog gmail search "in:anywhere" --max=100 --account=mitomi.yuuto2003@gmail.com
```

### 重要メールの抽出条件

最大100件のメールから、以下の条件で**特に重要なメール**を抽出してください：

**優先度 High（必ず通知）:**
- 件名に「緊急」「重要」「urgent」「important」「Action Required」が含まれる
- 人間からの個別メール（特定の個人名や個別のやりとり）
- 会議の招待や日程調整
- 請求書・領収書（ただし自動送信の購入レシートは除く）

**優先度 Medium（条件付きで通知）:**
- 今日または昨日の日付で、かつ未読のメール
- 返信が必要そうなメール

**除外（通知不要）:**
- 広告・プロモーションメール（SHEIN、ReFa、HAIR SALON など）
- 自動送信の購入レシート（Xなど）
- ニュースレター
- SNSの通知メール

### 通知条件

優先度 High のメールが **1件でもある場合** に通知してください。

### 出力形式

作業過程を記述した後、**必ず`## Response`セクションを作成**し、その下にユーザーへの最終通知内容を記述してください。

通知が必要な場合：

```markdown
## Response

📧 Gmail 未読メール: X件

重要なメール:
- [送信者] 件名（日時）
- [送信者] 件名（日時）

確認が必要です。
```

**注意**:
- 作業過程や分析は`## Response`の**前**に自由に記述してかまいません
- `## Response`セクションには、上記フォーマットに従った最終結果のみを記述してください
- メールの件名が英語の場合でも、説明文や「確認が必要です」などのメッセージは必ず日本語で記述してください

## チェック完了

すべてのチェックが問題なしの場合、`## Response`セクションに以下の文字列を記述してください：

```markdown
## Response

HEARTBEAT_OK
```

**重要**: 問題がない場合は、作業過程を記述した後、`## Response`セクションに`HEARTBEAT_OK`とだけ記述してください。
